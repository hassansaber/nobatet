## ۱. معرفی و چشم‌انداز محصول

**نوبتت** یک پلتفرم SaaS چندمستأجری (Multi-Tenant) نوبت‌دهی است که به کسب‌وکارهای ایرانی (کلینیک، آرایشگاه، مطب، سالن زیبایی، باشگاه و...) اجازه می‌دهد بدون نیاز به داشتن وب‌سایت اختصاصی، یک صفحه‌ی حرفه‌ای رزرو آنلاین + پنل مدیریتی + CRM داشته باشند. هدف اصلی پلتفرم **نوبت‌دهی هوشمند و بدون تداخل زمانی** است؛ همه‌ی امکانات دیگر (CRM، گزارش‌ها، باشگاه مشتریان و...) در خدمت همین هدف اصلی هستند.

### اصول طراحی محصول

- **Mobile First** در تمام صفحات (کاربر ایرانی عمدتاً از موبایل رزرو می‌کند)
- **RTL کامل** و تایپوگرافی فارسی (فونت پیشنهادی: Vazirmatn یا مشابه)
- **بدون نیاز به ایمیل** — احراز هویت صرفاً بر پایه‌ی شماره موبایل ایرانی
- **تقویم شمسی (جلالی)** در تمام بخش‌های تاریخ/زمان
- انیمیشن‌های ظریف و مدرن (Framer Motion یا مشابه)، ولی بدون افت پرفورمنس
- Production-ready، ماژولار، قابل توسعه

## ۲. نقش‌های کاربری (User Roles)

سیستم باید تک‌کالکشن/تک‌جدول کاربران با فیلد `role` باشد تا یک شماره موبایل بتواند همزمان چند نقش داشته باشد (مثلاً یک نفر هم صاحب بیزنس باشد هم مشتریِ بیزنس دیگری).

| نقش | توضیح |
|---|---|
| **Super Admin** | مدیر کل پلتفرم؛ نظارت بر همه‌ی بیزنس‌ها، ویزیتورها، اشتراک‌ها، تراکنش‌ها |
| **Visitor (ویزیتور/بازاریاب)** | با لینک/کد اختصاصی، بیزنس جذب می‌کند و از اشتراک‌های فروخته‌شده کمیسیون می‌گیرد؛ لندینگ اختصاصی خودش را دارد |
| **Business Owner (صاحب کسب‌وکار)** | مالک یک یا چند بیزنس؛ دسترسی کامل به پنل مدیریتی بیزنس |
| **Staff / Employee (نیروی کاری)** | کارمند وابسته به یک بیزنس؛ دسترسی محدود به تقویم/رزروهای خودش |
| **Customer (مشتری نهایی)** | مصرف‌کننده‌ی نهایی که نوبت رزرو می‌کند؛ داشبورد شخصی برای مشاهده‌ی تاریخچه‌ی نوبت‌ها |

هر نقش داشبورد اختصاصی خودش را دارد. مدل دسترسی باید RBAC پیاده‌سازی شود و در Proxy/میدلور Next.js چک شود.

### فلوی ثبت‌نام/ورود (برای همه‌ی نقش‌ها)

احراز هویت کاملاً بر پایه‌ی شماره موبایل و دومرحله‌ای است:

1. کاربر شماره موبایل را وارد می‌کند.
2. کد تأیید (OTP) پیامک می‌شود.
3. کاربر کد را وارد و تأیید می‌کند.
4. **اگر کاربر جدید است:** بلافاصله بعد از تأیید کد، فرم تکمیلی نمایش داده می‌شود و نام، نام خانوادگی و رمز عبور از او گرفته می‌شود؛ سپس حساب کاربری ساخته می‌شود.
5. **اگر کاربر قبلاً ثبت‌نام کرده:** بعد از تأیید کد مستقیم وارد داشبورد مربوط به نقشش می‌شود.
6. برای ورودهای بعدی، کاربر می‌تواند با «شماره موبایل + رمز عبور» یا دوباره با «شماره موبایل + OTP» وارد شود (هر دو روش فعال باشد).

## ۳. معماری چندمستأجری و Subdomain Routing

- هر بیزنس و هر ویزیتور یک ساب‌دامین اختصاصی می‌گیرد (Wildcard Subdomain)، مثلاً:
  - `dr-eghbali.business.nobatet.com`
  - `ali-rezaei.visitor.nobatet.com`
- کاربر نهایی باید حس کند این دامین متعلق به همان کسب‌وکار است (White-labeled feel).
- پیاده‌سازی از طریق **Next.js Proxy** (`src/proxy.js`) برای تشخیص ساب‌دامین و rewrite به روت داخلی مربوطه `/sites/[type]/[slug]/...` (توجه: پوشه‌ی `_sites` در App Router خصوصی/غیرقابل‌روت است).
- هر تنانت باید بتواند:
  - تم رنگی خودش را انتخاب کند
  - فیچرهای فعال روی لندینگ خودش را انتخاب/غیرفعال کند
  - لوگو، بنر، متن معرفی اختصاصی داشته باشد
- (فاز آینده) امکان اتصال دامنه‌ی اختصاصی (Custom Domain) برای بیزنس‌های پلن بالا.

## ۴. استک فنی (Tech Stack)

> **مهم: هنگام نصب پکیج‌ها هیچ ورژن خاصی را pin نکن؛ آخرین نسخه‌ی پایدار هر پکیج نصب شود.**

- **فریم‌ورک:** Next.js (App Router) + React
- **زبان:** JavaScript سرتاسری (بدون TypeScript)
- **استایل:** Tailwind CSS
- **دیتابیس:** PostgreSQL (میزبانی روی Liara)
- **ORM:** Drizzle ORM
- **احراز هویت:** JWT با httpOnly Cookie
- **پیامک:** ملی‌پیامک (SOAP Web Service)
- **پرداخت:** دو روش موازی — درگاه آزمایشی/Sandbox + کارت‌به‌کارت با تأیید دستی
- **هاستینگ:** Liara
- **PWA:** پیکربندی کامل
- **انیمیشن:** Framer Motion
- **Next.js 16:** Middleware renamed to **Proxy**

## ۵. قوانین حیاتی موتور رزرو (Booking Engine)

1. **قفل موقت زمان (Slot Locking):** به‌محض ورود مشتری به درگاه پرداخت، تایم انتخابی برای مدت مشخص (مثلاً ۱۰ دقیقه) قفل موقت می‌شود.
2. **آزادسازی خودکار:** اگر پرداخت در بازه‌ی زمانی تعیین‌شده تکمیل نشود، قفل به‌صورت خودکار آزاد شود.
3. **قطعیت رزرو فقط پس از پرداخت موفق:** هیچ رزروی بدون تأیید پرداخت (یا تأیید دستی ادمین برای رزرو حضوری) در تقویم نهایی ثبت نمی‌شود.
4. **جلوگیری از تداخل زمانی:** برای هر کارمند (Staff)، امکان دو رزرو هم‌زمان در یک بازه وجود ندارد — مگر خدمت از نوع «ظرفیت‌دار» باشد.
5. **بافر زمانی (Buffer Time):** خدمت ۶۰ دقیقه + بافر ۱۵ دقیقه = اولین تایم آزاد بعدی ۷۵ دقیقه پس از شروع.
6. **محاسبه‌ی تایم‌های آزاد:** ساعات کاری + مرخصی‌ها + مدت خدمت + بافر + رزروهای قبلی.
7. **قوانین لغو/تغییر:** هر بیزنس سیاست خودش را تعریف می‌کند و در تأیید رزرو به مشتری نمایش داده می‌شود.
